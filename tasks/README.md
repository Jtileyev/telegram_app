## Приоритеты

### 🔴 Критический (исправить немедленно)
Проблемы безопасности и функциональности, блокирующие деплой:
1. Хардкоженные пути в скриптах
2. Токен бота в базе данных
3. Прямые SQL запросы в хендлерах
4. Дефолтные учетные данные

**Оценка времени**: 5 часов
**Должно быть исправлено**: В течение 1-2 дней

### 🟠 Высокий (до продакшена)
Необходимо для безопасного запуска в продакшене:
1. Недоработанные фичи
2. HTTPS конфигурация
3. Система логирования
4. Резервное копирование БД
5. Rate limiting

**Оценка времени**: 2 недели
**Должно быть исправлено**: До продакшен деплоя

### 🟡 Средний (качество кода)

2. Пагинация
3. Локализация
4. CSRF защита
5. N+1 оптимизация
6. Валидация данных

**Оценка времени**: 1 месяц
**Должно быть исправлено**: В течение 2-3 месяцев

### 🟢 Низкий (улучшения)

1. Стиль кода

3. Redis для FSM

5. Очистка неиспользуемых полей
6. Покрытие тестами
7. CI/CD

**Оценка времени**: 2 месяца
**Можно отложить**: После запуска продакшена

2. Указать дату начала/завершения

4. Обновить оценку времени если нужно

## Важные правила

1. **Не пропускать критические задачи** - они блокируют все остальное
2. **Тестировать после каждого изменения** - запускать `./run_all_tests.sh`
3. **Делать маленькие коммиты** - каждая задача = отдельный коммит
4. **Обновлять документацию** - при изменении функционала

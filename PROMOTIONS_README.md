# Система Промо-Акций - Документация

## Обзор

Система промо-акций реализует программу лояльности типа "N-ое заселение — X бесплатных суток".

## Возможности

### Для администратора
- ✅ Создание и редактирование акций
- ✅ Включение/отключение акций
- ✅ Назначение акций к квартирам
- ✅ Просмотр статистики использования акций
- ✅ Отслеживание применённых бонусов в заявках

### Для пользователя
- ✅ Просмотр условий акции в карточке квартиры
- ✅ Отслеживание прогресса (сколько заселений осталось до бонуса)
- ✅ Автоматическое применение бонуса при достижении N-го заселения
- ✅ Уведомление об экономии при получении бонуса

## Использование

### Создание акции

1. Зайдите в админ-панель
2. Откройте раздел "Акции"
3. Нажмите "Добавить акцию"
4. Заполните поля:
   - **Название**: Краткое название (например, "Акция 6+1")
   - **Описание**: Подробное описание для пользователей
   - **Количество заселений до бонуса**: На каком заселении применяется бонус (N)
   - **Количество бесплатных дней**: Сколько дней дарится (X)
   - **Активна**: Включить/выключить акцию
5. Сохраните

### Назначение акции квартире

1. Откройте раздел "Квартиры"
2. Выберите квартиру для редактирования
3. В поле "Программа лояльности (Акция)" выберите нужную акцию
4. Сохраните

### Примеры акций

#### Акция "6+1"
- **Условие**: 6 заселений
- **Бонус**: 1 бесплатный день
- **Описание**: На каждом 6-ом заселении пользователь получает 1 бесплатный день

#### Акция "3+2"
- **Условие**: 3 заселения
- **Бонус**: 2 бесплатных дня
- **Описание**: На каждом 3-ем заселении пользователь получает 2 бесплатных дня

## Как работает система

### Механика

1. **Пользователь бронирует квартиру** с активной акцией
2. **Система считает заселения** только для пары (пользователь + квартира)
3. **При достижении N-го заселения**:
   - Автоматически применяется бонус (X бесплатных дней)
   - Пересчитывается стоимость: `итого = (общие дни - бесплатные дни) × цена за день`
   - Пользователь видит уведомление об экономии
4. **После завершения бронирования** с бонусом:
   - Цикл обнуляется
   - Начинается новый цикл с 1-го заселения

### Учёт заселений

- ✅ Учитываются только **завершённые** бронирования (status = 'completed')
- ✅ Каждая пара (пользователь + квартира) имеет свой прогресс
- ✅ Прогресс сохраняется между бронированиями
- ✅ После применения бонуса прогресс обнуляется

## API / Функции базы данных

```python

# Получить прогресс пользователя

# Рассчитать бонус

    user_id, apartment_id, num_days

# Применить бонус к бронированию

    booking_id, promotion_id, discount_days, original_price

# Обновить прогресс при завершении бронирования

```

## Тестирование

### Сценарий 1: Обычное заселение
1. Создайте акцию "6+1" (6 заселений → 1 день)
2. Назначьте акцию квартире
3. Создайте бронирование от пользователя A
4. В админке увидите прогресс: 0/6
5. Завершите бронирование (status = 'completed')
6. Прогресс станет: 1/6

### Сценарий 2: Бонусное заселение
1. У пользователя A уже 5 завершённых заселений
2. Создайте новое бронирование на 10 дней, цена 15 000 ₸/день
3. Система применит бонус: -1 день
4. Итоговая цена: 9 дней × 15 000 = 135 000 ₸
5. Пользователь увидит уведомление об экономии 15 000 ₸
6. После завершения прогресс обнулится: 0/6

### Сценарий 3: Новый цикл
1. После бонуса пользователь A снова начинает с 1-го заселения
2. Прогресс: 0/6 → 1/6 → 2/6 → ... → 5/6 → бонус → 0/6

## FAQ

**Q: Можно ли назначить несколько акций одной квартире?**
A: Нет, квартира может иметь только одну активную акцию.

**Q: Что если изменить параметры акции?**
A: Изменения применятся к новым бронированиям. Существующий прогресс пользователей сохранится.

**Q: Можно ли удалить акцию?**
A: Можно удалить только неиспользуемые акции (не назначенные ни одной квартире).

**Q: Учитываются ли отменённые бронирования?**
A: Нет, учитываются только завершённые (status = 'completed').

**Q: Что если бесплатных дней больше, чем дней бронирования?**
A: Система автоматически ограничит бонус количеством дней бронирования.
